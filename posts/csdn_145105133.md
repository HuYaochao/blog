 在Java中，
**动态绑定**（Dynamic Binding）和
**动态分派**（Dynamic Dispatch）是两个相关但不同的概念。

---

## 1.动态绑定（Dynamic Binding）

**定义**  
 动态绑定是指在运行时决定方法的具体实现绑定到哪一个方法体。也可以称为**后期绑定**（Late Binding）。动态绑定主要适用于**多态**的场景。

**特点**

- 动态绑定发生在方法调用时，而不是在编译时。
- 适用于**实例方法**，不适用于静态方法和成员变量。
- JVM根据对象的**实际类型**（运行时类型）来选择方法实现，而不是引用的编译时类型。

**示例**

```
class Animal {
    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Dog(); // 编译时类型是 Animal，运行时类型是 Dog
        myAnimal.sound(); // 输出：Dog barks
    }
}
```

在这个例子中，`myAnimal`的编译时类型是`Animal`，但由于动态绑定，运行时调用了`Dog`类的`sound`方法。

---

## 2.动态分派（Dynamic Dispatch）

**定义**  
 动态分派是指在运行时根据对象的实际类型选择方法的过程。这是动态绑定背后的机制。动态分派专门用于**方法重写**（Method Overriding）的场景。

动态分派机制是通过虚方法表（**Virtual Method Table，V-Table**）实现的。每个类都会有一张虚方法表，记录该类的每个方法实现的入口地址。JVM通过对象的实际类型找到对应的方法实现。

**特点**

- 动态分派是动态绑定的实现机制。
- 仅针对**实例方法**。
- 依赖于继承和方法的重写。

**示例**  
 与动态绑定的示例一致，动态分派的核心在于 JVM 如何找到 `Dog` 类的 `sound` 方法实现。

## 3.区别

| **特性** | **动态绑定** | **动态分派** |
| --- | --- | --- |
| **概念范围** | 是一种行为：方法调用时在运行时绑定到具体方法体 | 是一种机制：通过对象的实际类型选择正确的方法实现 |
| **适用场景** | 动态绑定是运行时绑定的结果 | 动态分派是多态实现的基础 |
| **作用对象** | 适用于所有运行时绑定的情况（如多态方法） | 专注于实例方法的重写（override） |
| **实现机制** | JVM 在运行时绑定方法实现 | JVM 通过虚方法表选择具体的实现 |

---

## 总结

动态绑定是运行时选择方法实现的行为，而动态分派是实现动态绑定的核心机制之一。简单来说，动态分派是动态绑定的技术支持，二者是概念上的紧密关联。
