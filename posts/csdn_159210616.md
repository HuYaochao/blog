2026 年 1 月，一个叫 OpenClaw 的开源项目以 72 小时斩获 60,000 颗 Star 的速度席卷 GitHub。100 天后，它以 250,000+ Star 超越 React 13 年的积累，成为 GitHub 上 Star 数最高的软件项目。NVIDIA CEO 黄仁勋在 GTC 2026 上说："每家公司都需要一个 OpenClaw 战略。“深圳龙岗区连夜推出"龙虾十条”，最高补贴 200 万。腾讯大厦楼下近千人排队免费装机。

这是一个放大器被当成了发动机的故事。

---

### 一、耗散结构：看起来活着，不代表是生命

物理学里有一类系统叫**耗散结构**（Dissipative Structure），它离热力学平衡态很远，靠持续消耗外部能量来维持自身的有序。飓风是典型案例——它有结构、有运动、能自我维持、甚至能"成长"。但它不是生命。切断海水热能，它就消散了。

OpenClaw 就是 AI 领域的飓风。

它 7×24 运行、有持久记忆、能自主拆解任务、跨平台执行操作。它的所有记忆、配置、任务记录以 Markdown 和 YAML 纯文本格式保存在本地——文件即状态，隐私可控，看上去像一个"数字生命体"。但深入一层就会发现：它存储的是状态，不是智能。每一次任务执行中的"智力"，都是从底层大语言模型的 API 调用中临时借来的。

OpenClaw 自己的技术文档也承认这一点：复杂任务的执行上限，受制于后端 LLM 的智力水平，尤其是长程规划能力。更直白地说，OpenClaw 本质上是一个"智能体网关"——繁重的 AI 推理仍然依赖云端模型服务器，本地设备负责的是收发消息、调用 API 和运行简单脚本。

飓风的大小取决于海水温度，不取决于气旋的形状。OpenClaw 的能力上限取决于底层模型的推理质量，不取决于 Agent 框架的架构设计。

需要精确一点：这不是说 Agent 框架毫无贡献。OpenClaw 的重试逻辑、状态持久化、沙箱隔离、人机审批节点——这些都是模型本身不具备的**可靠性工程**。同一个底层模型接入不同 Agent 框架，端到端成功率确实存在显著差异。所以 OpenClaw 不是被动的温度计——它是一个有增益的放大器。但放大器的输出天花板，仍然由输入信号的质量决定。一个增益 10 倍的放大器，输入信号是噪声，放大出来的还是噪声。

---

### 二、乘法栈：放大器放大的是什么

#### 2.1 先理解正向的乘法效应

GTC 2026 的核心技术叙事，是 LLM 推理降本的六层技术栈——从硅片架构到工程调度，六个独立的优化层各自贡献 2-10 倍的成本压缩，叠加效果是乘法而非加法。保守估计综合降本 216 倍，激进估计超过 7,000 倍。Epoch AI 观测到的实际数据支持这个判断：2024 年初到现在，某些基准测试上的推理价格已经下降了近三个数量级。

这个乘法逻辑同样适用于能力的传递。**多模态大模型 → AI Agent → 具身智能**，不是一层层加上去的，而是一层层乘上去的：

| 层级 | 它乘的是什么 | 乘数来源 |
| --- | --- | --- |
| 多模态大模型 | 基准智能——推理、规划、工具理解、多模态感知 | 模型架构 + 训练数据 + 算力 |
| AI Agent | 工具调用可靠性 × 上下文管理 × 多步链稳定性 | 框架工程 + prompt 设计 |
| 具身智能 | 物理反馈延迟 × 安全约束 × 环境泛化 | 传感器 + 控制算法 + 世界模型 |

当底层模型是世界顶尖时，乘法效应产生指数级的能力爆发：

```
模型基准 1.0 × Agent 0.9 × 具身 0.85 = 0.77（可用）
```

#### 2.2 乘法的残酷：退化也是乘法

但乘法不长眼睛。它放大优势，也放大缺陷：

```
模型基准 0.7 × Agent 0.7 × 具身 0.7 = 0.34（废了）
```

0.7 是什么概念？2025-2026 年的实测数据给出了具体画面。

**长上下文退化（Context Rot）**：斯坦福团队发表的"Lost in the Middle"研究表明，当关键信息位于长上下文的中间位置时，LLM 的检索准确率可下降超过 30%，呈现 U 型性能曲线——开头和结尾的信息利用率远高于中间。2025 年的后续研究进一步发现，当上下文窗口使用率超过 50% 时，U 型曲线本身也会退化，模型开始偏向最近的 token，丢弃早期信息。更关键的是，这不是检索失败——研究者通过实验证明，即使把所有非关键 token 替换为空格，性能仍然随输入长度下降。长度本身就是毒药。

**多步工具调用的复合错误**：OSWorld 50 步挑战——模拟一个需要连续执行 50 个 GUI 操作的计算机任务——当前最先进的 AI Agent 成功率仅为 34.5%。也就是说，在一个需要连续做对 50 件事的场景里，三分之二的尝试以失败告终。Google DeepMind 的大规模研究发现，当 Agent 数量超过 4 个时，准确率增益开始饱和甚至下降——多智能体协作引入的"协调税"会吞噬掉能力增长。更直观的行业数据是：某金融科技公司的 AI Agent 在交易对账中进入死循环，运行了 11 天，烧掉 47,000 美元才被发现。

**这就是乘法的残酷**：Agent 框架再精巧，如果底层模型在长上下文中的可靠性是 0.7，在复杂工具链中的稳定性也是 0.7，那最终的端到端成功率就是 0.7 × 0.7 = 0.49。再加一层具身智能的物理世界反馈，0.49 × 0.7 = 0.34。你得到的不是一个"AI 员工"，而是一个成功率三分之一的赌局。

### 三、语言是多模态的骨架

说"大模型是地基"还不够精确。更准确的表述是：**多模态大模型是地基，而语言是多模态的骨架**。

为什么语言优先？因为推理、规划、工具调用——Agent 框架中所有需要"思考"的环节——全部通过语言通道完成。视觉和音频是感知层，它们告诉模型"看到了什么"和"听到了什么"；但"接下来该做什么"、“哪些步骤要先做”、“这个工具的返回值意味着什么”——这些全是语言。

一个瞎子可以推理。一个不能推理的"眼睛"什么也做不了。

具身智能更是如此。人形机器人需要理解人类指令（语言）、分解任务步骤（推理）、感知环境（视觉 + 触觉）、执行操作（控制）——其中感知和控制可以通过传感器和电机实现，但"理解指令"和"分解步骤"的质量完全取决于语言大模型的推理能力。

行业共识也在收敛到这个判断。月之暗面的杨植麟直言："解决长尾复杂任务的根本能力，仍深植于模型层本身。解决它需要回到模型层，烧卡训练。"所以模型公司正在不可避免地变成产品公司——千问、智谱、Kimi，都在将模型能力直接封装为可用的 Agent 服务。因为他们知道，Agent 的天花板不在 Agent。

---

### 四、成本悬崖：乘法效应的另一面

GTC 2026 展示的降本技术栈，本质上是同一个乘法逻辑的另一面——如果六层优化各贡献 3 倍压缩，综合就是 729 倍。这意味着两件事：

**第一，顶尖模型的推理成本正在以年化 10 倍以上的速度下降**。Rubin GPU（2026 H2 量产）相比 Blackwell 推理吞吐量每瓦提升 10 倍，MLA（多头潜在注意力）将 KV Cache 压缩 93% 以上，Prompt Caching 在重复前缀场景下节省 72% 以上计算量。NVIDIA 首次在硬件层面实体化了分离式推理——Rubin GPU 处理 Prefill（算力密集），Groq 3 LPU 处理 Decode（带宽密集），各司其职。

**第二，成本下降使"顶尖模型 + Agent 框架"的组合成为经济上可行的默认选项**。以前用顶尖模型跑 Agent 太贵，只能凑合用小模型——这就是在乘法栈的第一个乘数上打折。现在成本悬崖意味着：你不需要在模型质量上妥协。

这就形成了一个残酷的竞争格局：

```
最优解 = 世界顶尖模型 × 最成熟 Agent 框架 × 最低推理成本
       = 乘法效应最大化
```

如果你的模型不是世界顶尖，你在乘法栈的第一个位置就输了。后面的 Agent 框架再好、部署成本再低，乘出来的结果都不如对手。

### 五、第一乘数的投入门槛：全球化不是选项，是前提

前四节确立了一个判断：乘法栈的第一乘数——多模态大模型的推理质量——决定了整个栈的上限。

这一节要问的问题是：**维持第一乘数的世界顶尖水平，需要什么？**

答案是：需要持续的、天文数字级别的资本投入，而回收这种量级的投入，仅靠单一市场不可能。

数据已经说明了这个逻辑。OpenAI 在 2026 年 2 月达到 250 亿美元年化收入，Anthropic 达到 190 亿美元。但两家公司都在持续烧钱——OpenAI 预计 2026 年现金消耗 170 亿美元，Anthropic 同样以数十亿美元级别燃烧。美国在 2025 年捕获了全球 AI 投资的 79%（1,590 亿美元），旧金山湾区一地就占了美国 AI 投资的 76%。即使是世界上最大的 AI 市场，内部的价值分布也极度集中。

AI 替代的主要是高脑力、高工资、高标准的服务业——编程、法律分析、金融研报、医疗影像、客服中台。这些岗位的价值密度高，AI 替代后的经济效益最显著。但任何单一国家中此类就业的绝对规模，都不足以支撑万亿级 AI 收入。

这意味着什么？**第一乘数的竞争本质上是一场资本军备竞赛，而军备竞赛的弹药来自全球收入。** 模型层的竞争从第一天起就是全球竞争——不是因为"国际化是好的商业策略"，而是因为如果你不能从全球市场获取高价值客户来回收训练和推理基础设施的投入，你就无法持续在第一乘数上保持顶尖。

这就把全球化从一个"商业选择"变成了第一乘数的**结构性前提**。那些只在本土市场打转的模型厂商，面临的不是"增长放缓"的问题，而是一个更根本的困境：收入天花板低于保持第一乘数竞争力所需的持续投入。乘法栈的残酷逻辑不只作用于技术层面——它同样作用于商业层面。第一乘数不够强，技术上输；收入不够多，商业上无法维持第一乘数。这是同一个循环的两面。

### 六、耗散结构不会自己变成生命

回到 OpenClaw。

它的爆火不是偶然——它精准踩中了一个需求：人们受够了只动嘴不动手的 AI，渴望能真正执行任务的工具。从技术架构上看，它的"网关-节点-渠道"三层解耦、MCP 协议兼容、纯文本状态管理，都是扎实的工程设计。NVIDIA 推出的 NemoClaw（企业级 OpenClaw）更是证明了这个方向的产业价值。

但 OpenClaw 生态暴露出的问题，恰恰验证了"乘法栈第一乘数决定天花板"的判断：

- **ClawHub 技能市场 12% 的恶意率**——341 个恶意 Skill 被安全公司发现。这不是 Agent 框架的问题，而是底层模型无法可靠识别伪装恶意代码的问题。
- **ClawJacked 安全漏洞**——攻击者通过恶意网页就能接管 AI 智能体。这不是沙箱隔离的问题，而是 LLM 对 prompt 注入攻击缺乏根本免疫力的问题。
- **"开箱即崩"的用户体验**——90% 的公网部署实例可被绕过身份验证。框架层面可以补课，但根本问题是 LLM 在安全推理上的不可靠。

OpenClaw 的创始人 Peter Steinberger 自己也承认：“提示注入攻击仍是行业尚未解决的难题。”

**耗散结构不会自己变成生命。** 飓风不会因为持续旋转就进化出自我修复能力。OpenClaw 不会因为持续运行就解决底层模型的推理缺陷。Agent 框架可以把模型的能力组织得更好、放大得更远——好的放大器确实能提升信噪比、减少失真、增加增益。但放大器不生产信号。当信号本身不够强时，再好的放大器也只是在放大局限。

---

### 七、结论：一个判断，一个推论，一个边界

**判断**：多模态大模型是 AI 产业乘法栈的第一个乘数。Agent 框架和具身智能是后续乘数。第一个乘数决定整个栈的上限——不是"之一"，是唯一的第一优先级。

**推论**：对于试图在模型层竞争的头部厂商，在大模型层达到世界顶尖之前，任何在 Agent 和具身智能上的投入都是在"放大器增益"上做文章。增益再高，输入信号不够强，输出就起不来。而维持世界顶尖的模型能力，需要全球化收入作为持续投入的燃料——这不是商业偏好，是结构性前提。

**边界**：这个判断适用于那些选择在第一乘数上竞争的玩家。对于深耕特定垂直场景的 Agent 开发者、做可靠性工程的框架团队、或在具身智能领域有独特传感器和控制能力的硬件厂商，他们的比较优势在乘法栈的其他位置——在那些位置上做到极致，同样有价值。乘法栈是乘法，每个乘数都在乘。只是第一个乘数设定了天花板。

OpenClaw 是这个时代最好的信号放大器。它放大出来的信号告诉我们：底层模型正在变强，但还没有强到让 Agent 框架自持的程度。

对于在第一乘数上竞争的人，最聪明的做法是继续增强信号源——而不是去优化放大器的增益曲线。
