## 一、更换yum镜像源

### 1.1 更新或替换 Yum 仓库配置

由于 CentOS 7 官方仓库已经进入维护状态，其默认配置可能无法满足当前需求。建议使用国内镜像源以获得更高的稳定性与下载速度。请尝试以下步骤：

- **备份现有仓库配置：**

  ```
  sudo mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.bak
  ```
- **下载新的 Aliyun 源配置：**

  ```
  sudo curl -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
  ```
- **清理缓存并生成新的缓存：**

  ```
  sudo yum clean all && sudo yum makecache
  ```


您可以先备份当前的仓库配置，然后删除所有现有的 `.repo` 文件，再重新添加新的仓库配置文件。以下是详细步骤和命令示例，供企业级环境中参考和实施：

> 1. 备份现有的仓库配置

在进行任何删除操作前，务必先备份 `/etc/yum.repos.d/` 目录，以便在需要时快速恢复：

```
sudo cp -r /etc/yum.repos.d /etc/yum.repos.d.backup
```

> 2. 删除所有现有的 `.repo` 文件

删除该目录下所有的仓库配置文件：

```
sudo rm -f /etc/yum.repos.d/*.repo
```

> 3. 添加新的仓库配置

以国内的 Aliyun 源为例，重新下载并添加新的仓库配置文件：

```
sudo curl -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
```

完成后，可执行以下命令清理缓存并生成新的仓库索引：

```
sudo yum clean all && sudo yum makecache
```

## 二、yum安装nginx

在企业环境中，使用 Yum 安装 Nginx 可以分为以下步骤，以确保系统安全、稳定和高效：

> 1. 启用 EPEL 仓库

Nginx 通常不包含在默认的 CentOS 仓库中，因此首先需要启用 EPEL（Extra Packages for Enterprise Linux）仓库：

```
sudo yum install -y epel-release
```

启用 EPEL 仓库后，可以获取更多经过测试且维护良好的软件包，符合企业级需求。

> 2. 安装 Nginx

启用 EPEL 仓库后，直接使用 Yum 命令安装 Nginx：

```
sudo yum install -y nginx
```

此命令将自动解决依赖关系，并从官方或镜像源中获取最新稳定版本的软件包。

> 3. 配置与启动服务

安装完成后，建议立即启动并设置 Nginx 开机自启，以确保服务在系统重启后自动恢复：

```
sudo systemctl start nginx
sudo systemctl enable nginx
```

此外，可使用以下命令检查服务状态，确保 Nginx 正常运行：

```
sudo systemctl status nginx
```

> 4. 防火墙配置

在企业级环境中，安全配置至关重要。确保防火墙已允许 HTTP/HTTPS 流量：

```
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

这样能够确保外部访问不受阻碍，同时保持整体安全性。

## 三、部署静态页面

> 1. 修改 Nginx 配置

打开 Nginx 主配置文件：

```
sudo vi /etc/nginx/nginx.conf
```

找到 `http {` 段落，在 `server {}` 之外，添加一个新的 `server` 配置块：

```
server {
    listen 80;
    server_name _;  # 监听所有请求，也可替换为你的域名或IP

    root /usr/local/test;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    # 允许跨域访问（可选）
    location /api/ {
        add_header Access-Control-Allow-Origin *;
    }
}
```

**保存并退出**：

1. 按 `ESC`
2. 输入 `:wq`
3. 回车

---

> 2. 检查 Nginx 配置

在应用新配置前，先检查 Nginx 语法是否正确：

```
sudo nginx -t
```

如果输出：

```
nginx: configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
```

说明配置正确，可以继续。

---

> 3. 重新加载 Nginx

```
sudo systemctl restart nginx
```

或者：

```
sudo nginx -s reload
```

---

> 4. 开放防火墙端口

如果服务器启用了 `firewalld`，运行：

```
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```

如果使用 `iptables`：

```
sudo iptables -I INPUT -p tcp --dport 80 -j ACCEPT
sudo service iptables save
```

---

> 5. 访问网站

在浏览器中输入：

```
http://your_server_ip/
```

如果 `index.html` 页面能正常加载，说明部署成功。

---

> 6. 设置 Nginx 开机自启

```
sudo systemctl enable nginx
```

---

> 可选优化

1. **日志监控**  
    查看 Nginx 访问日志：

   ```
   sudo tail -f /var/log/nginx/access.log
   ```

   查看错误日志：

   ```
   sudo tail -f /var/log/nginx/error.log
   ```

现在，你的 Nginx 服务器已经成功挂载静态网页 🎉🚀，可以访问 `http://your_server_ip/` 进行测试！
