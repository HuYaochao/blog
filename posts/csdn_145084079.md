## 1. Nginx 简介

Nginx (Engine X) 是一个开源的高性能 HTTP 服务器和反向代理服务器，同时也支持 IMAP/POP3 邮件代理。它以**轻量级、稳定性高、资源占用少、并发能力强**而闻名，广泛应用于 Web 服务和分布式系统中。

- **核心特性**：
  - **静态资源服务**：高效提供 HTML、CSS、JavaScript 等资源。
  - **反向代理**：代理用户请求到后端服务。
  - **负载均衡**：支持多种策略分发请求。
  - **模块化扩展**：支持 gzip 压缩、缓存、限流等功能。

## 2. 安装与配置：快速起步

### 2.1 下载与安装

- **下载地址**：[Nginx 官方站点](https://nginx.org/en/download.html)
- **安装步骤**：
  1. 下载 Windows 版 Nginx 压缩包。
  2. 解压到目标目录，例如 `C:\nginx`。
  3. 打开命令提示符，进入目录并运行：

     ```
     start nginx
     ```

### 2.2 常用命令

以下是 Nginx 的启动、停止和重启方法：

- **启动 Nginx**：


## 3. 实践：部署简单的静态网站

index.html示例代码

```
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Custom HTML</title>
</head>
<body>
    <h1>Hello from Custom HTML</h1>
</body>
</html>
```

### 3.1 配置文件示例

以下为简单的 Nginx 配置，用于静态 HTML 网站的托管：

```
# 核心配置
user  nobody;               # 指定运行用户，Windows 中无实际作用
worker_processes  1;        # 工作进程数量，1 表示单线程

events {
    worker_connections  1024; # 每个进程最大连接数
}

http {
    include       mime.types;        # 包含 MIME 类型映射
    default_type  application/octet-stream;  # 默认 MIME 类型
    sendfile        on;              # 开启高效文件传输模式
    keepalive_timeout  65;           # 连接保持时间

    server {
        listen       80;              # 监听端口
        server_name  localhost;       # 服务器名称

        location / {
            root   C:/nginx/html/custom_site; # 静态资源目录
            index  index.html;                # 默认主页文件
        }

        error_page   500 502 503 504  /50x.html; # 定义错误页面
        location = /50x.html {
            root   html;
        }
    }
}
```

### 3.2 配置说明

- **worker\_processes**：指定 Nginx 工作进程数，建议设置为 CPU 核心数。
- **worker\_connections**：每个进程允许的最大连接数。
- **include mime.types**：定义不同文件类型的 MIME 映射。
- **location /**：匹配所有以 `/` 开头的 URL，定义其根目录和默认文件。
- **error\_page**：自定义错误页面，例如 `500`、`503`。

## 4. 进阶应用与学习方向

Nginx 的功能远不止静态资源托管，以下是常见的应用场景及学习建议：

### 4.1 反向代理与负载均衡

- **反向代理**：通过 Nginx 转发用户请求到后端服务（如 Java 应用服务器）。
- **负载均衡**：实现多实例分发，支持多种调度算法（如轮询、最少连接、IP 哈希）。
- **学习方向**：
  - 配置 `proxy_pass` 实现反向代理。
  - 学习负载均衡策略，如 `upstream` 配置。

### 4.2 安全与优化

- **HTTPS 支持**：使用 SSL/TLS 加密，确保数据传输安全。
- **请求限流**：限制高频访问，防止恶意攻击。
- **学习方向**：
  - 配置 HTTPS（申请并使用免费证书如 Let’s Encrypt）。
  - 研究 Nginx 限流模块。

### 4.3 模块化扩展

- **动态内容服务**：通过 FastCGI 或后端服务处理动态请求（如 PHP、Python）。
- **学习方向**：
  - 配置 PHP 环境，结合 FastCGI 提供动态服务。
  - 学习第三方模块扩展。

## 5. Nginx 目录结构解析

安装 Nginx 后，解压得到的目录结构如下所示。每个文件夹和文件都有其特定用途，理解这些内容可以帮助我们更高效地使用和管理 Nginx。

---

### 5.1 win主目录结构

```
│  nginx.exe                # Nginx 主程序
├─conf                      # 配置文件目录
├─contrib                   # 贡献的脚本和工具
├─docs                      # 文档与许可证信息
├─html                      # 默认静态资源目录
├─logs                      # 日志目录
└─temp                      # 临时文件目录
```

---

### 5.2 各子目录详解

**1. `nginx.exe`**

- **用途**：Nginx 的核心执行文件，通过此文件启动、停止或管理 Nginx。

---

**2. `conf`**

- **用途**：存放 Nginx 的配置文件。
- **主要文件**：
  - **`nginx.conf`**：
    - 主配置文件，定义全局参数、事件处理及 HTTP 服务配置。
  - **`mime.types`**：
    - 定义文件扩展名与 MIME 类型的映射关系。
    - 示例：`.html` 对应 `text/html`，`.jpg` 对应 `image/jpeg`。
  - **`fastcgi.conf`** 和 **`fastcgi_params`**：
    - 用于配置与 FastCGI 服务器交互的参数。
    - 示例：处理 PHP 时需要指定相关参数。
  - **`uwsgi_params`** 和 **`scgi_params`**：
    - 分别用于 uWSGI 和 SCGI 的参数配置。
  - **`koi-utf`**、**`koi-win`**、**`win-utf`**：
    - 字符编码映射文件，通常用于特定字符集转换场景。

---

**3. `contrib`**

- **用途**：存放 Nginx 社区贡献的工具和脚本，提供一些增强功能。
- **内容**：
  - **`geo2nginx.pl`**：
    - 用于将 GeoIP 数据转换为 Nginx 支持的格式。
  - **`vim`**：
    - 为 Vim 编辑器提供 Nginx 配置文件的语法高亮支持。
  - **`unicode2nginx`**：
    - 工具脚本和配置文件，用于处理 Unicode 编码映射。

---

**4. `docs`**

- **用途**：存放 Nginx 的文档和许可证信息。
- **主要文件**：
  - **`README`**：
    - 简要说明 Nginx 的功能及使用方法。
  - **`LICENSE`**、**`OpenSSL.LICENSE`** 等：
    - Nginx 及其依赖组件的许可证信息。
  - **`CHANGES`** 和 **`CHANGES.ru`**：
    - Nginx 的变更日志，分别为英文和俄文版本。

---

**5. `html`**

- **用途**：默认的静态资源存放目录。
- **主要文件**：
  - **`index.html`**：
    - 默认主页文件，Nginx 启动后访问 `http://localhost` 时加载此文件。
  - **`50x.html`**：
    - 用于显示 500 系列服务器错误页面。

---

**6. `logs`**

- **用途**：存放 Nginx 的运行日志。
- **主要文件**：
  - **`access.log`**：
    - 记录所有访问请求，包括 IP 地址、请求时间、请求路径等信息。
  - **`error.log`**：
    - 记录运行过程中产生的错误信息。

日志文件可以用来排查问题，例如查看访问是否正常，或者定位具体的错误原因。

---

**7. `temp`**

- **用途**：存放临时文件，用于处理客户端请求时的缓存和中转。
- **子目录**：
  - **`client_body_temp`**：
    - 存放客户端上传数据的临时文件。
  - **`proxy_temp`**、**`fastcgi_temp`**、**`scgi_temp`**、**`uwsgi_temp`**：
    - 分别用于反向代理、FastCGI、SCGI 和 uWSGI 的临时数据存储。

这些临时目录对高并发和大文件上传等场景非常重要，默认会自动清理，但也可以根据需求进行优化。

---

### 5.3 Linux 环境下的 Nginx 目录结构

在 Linux 系统上安装的 Nginx 目录结构与 Windows 上有所不同，主要是由于两者的运行环境差异以及部署方式的不同。下面我们详细解析 Linux 环境下的 Nginx 文件和目录结构，并对比两者的不同。  
 以典型的源代码解压目录为例，结构如下：

```
auto/             # 自动生成配置文件的脚本和模板
conf/             # 默认的配置文件目录
contrib/          # 贡献工具与脚本
html/             # 默认的静态资源目录
man/              # 手册页
src/              # 源代码目录
CHANGES           # 版本更新日志
CHANGES.ru        # 俄文版更新日志
configure         # 配置编译参数的脚本
LICENSE           # Nginx 的许可证
README            # 简要说明
```

---

**1.** `auto/`

- **用途**：用于生成编译配置的脚本和模板文件。
- **内容**：
  - 包含与系统和编译环境相关的检测脚本。
  - 这些脚本用于确定系统支持的功能（如模块、依赖库等）。
- **使用**：
  - 当你运行 `./configure` 脚本时，会调用 `auto/` 目录下的脚本自动生成配置。

---

**2.** `conf/`

- **用途**：存放 Nginx 的默认配置文件，功能与 Windows 版的 `conf/` 相同。
- **主要文件**：
  - **`nginx.conf`**：主配置文件。
  - **其他配置文件**：如 MIME 类型映射文件、FastCGI 参数等。
- **注意**：通过源代码编译安装的 Nginx 可能默认没有太多额外的配置文件，取决于安装时的参数。

---

**3.** `contrib/`

- **用途**：社区贡献的脚本和工具，与 Windows 版本类似。
- **典型内容**：
  - **`geo2nginx.pl`**：将 GeoIP 数据转换为 Nginx 支持的格式。
  - **`vim`**：提供 Vim 编辑器的语法高亮支持。

---

**4.** `html/`

- **用途**：默认的静态资源存放目录，功能与 Windows 环境一致。
- **典型内容**：
  - `index.html`：默认主页。
  - `50x.html`：错误页面。

---

**5.** `man/`

- **用途**：存放 Nginx 的手册页。
- **主要内容**：
  - 用于查看 Nginx 的命令行使用帮助和参数说明。
  - 可通过 `man nginx` 查看相关内容。

---

**6.** `src/`

- **用途**：存放 Nginx 的源代码。
- **主要内容**：
  - 核心功能代码（如 HTTP、反向代理、模块化扩展等）。
  - 每个子目录对应 Nginx 的某个模块或功能。

---

**7.** `CHANGES` 和 `CHANGES.ru`

- **用途**：记录 Nginx 的更新日志。
- **区别**：
  - `CHANGES`：英文版。
  - `CHANGES.ru`：俄文版（由于 Nginx 是俄罗斯开发团队创建的）。

---

**8.** `configure`

- **用途**：用于配置 Nginx 的编译参数。
- **使用方法**：
  - 在编译 Nginx 之前，运行 `./configure` 指定所需模块或路径。
  - 示例：

    ```
    ./configure --prefix=/usr/local/nginx --with-http_ssl_module
    ```
  - 参数说明：
    - `--prefix`：指定 Nginx 的安装目录。
    - `--with-*`：启用特定模块（如 SSL 支持）。

---

**9.** `LICENSE`

- **用途**：Nginx 的开源许可证，通常是 BSD 协议。

---

**10.** `README`

- **用途**：简要说明 Nginx 的功能和构建方法。

---

### 5.4 对比：Linux 与 Windows 目录结构的不同

| **功能** | **Windows** | **Linux** |
| --- | --- | --- |
| **主程序** | `nginx.exe` | 编译后在 `sbin/nginx` |
| **配置文件目录** | `conf/` | `conf/` |
| **静态资源目录** | `html/` | `html/` |
| **日志目录** | `logs/` | 安装后可能在 `/var/log/nginx` |
| **临时目录** | `temp/` | `/var/cache/nginx` |
| **扩展功能** | `contrib/` | `contrib/` |
| **源码与工具** | 无 | `src/`、`auto/` |
| **手册页** | 无 | `man/` |

### 5.5 学习与实践建议

1. 理解配置文件

- 配置文件是 Nginx 的核心。重点关注 `nginx.conf`，熟悉每个指令的用途是掌握 Nginx 的关键。  
   例如，`server` 块定义虚拟主机，`location` 块指定请求路径的处理逻辑等。

  - **`error.log`**：记录服务运行中的错误信息，帮助快速定位问题。

  - **`client_body_temp`**：保存客户端请求体的临时文件。
  - **`proxy_temp`**：保存代理服务器的临时文件。

- 在 `contrib/` 目录中，有许多实用的工具和脚本：

- 学习如何结合这些工具与实际需求，增强 Nginx 的功能。

---

5. 了解安装方式

    ```
    sudo apt install nginx   # Ubuntu/Debian

    ```
  - 从源代码编译安装，适合需要定制化功能的场景。

---

6. 熟悉编译参数

- 在 Linux 上，从源代码编译 Nginx 可以灵活配置所需模块：
  - 示例：启用 HTTP/2 和 SSL 支持：

    ```
    ./configure --with-http_v2_module --with-http_ssl_module
    ```
- 通过合理的模块选择，可以减少不必要的功能，提升运行效率。

---

7. 深入源代码

- 对于有兴趣研究 Nginx 内部实现的人，可以分析 `src/` 目录下的代码：
  - 学习 Nginx 的事件驱动模型如何实现高性能。
  - 研究负载均衡和请求处理的核心逻辑。
- 这种深入学习有助于更好地理解 Nginx 的设计思想，并为定制开发奠定基础。

---

8. 掌握自动化部署

- 在 Linux 环境中，自动化部署是提高运维效率的重要手段：
  - 使用 `systemd` 实现开机自启：

    ```
    sudo systemctl enable nginx
    ```
  - 配合 CI/CD 工具（如 Jenkins、GitLab CI）实现自动化的配置管理和发布流程。

## 6. 全面解读 Nginx 配置文件

以下是 Nginx 的核心配置文件 `nginx.conf` 的全面解读，帮助开发者理解各部分的功能及其应用场景。

---

### 6.1 全局配置块

```
#user  nobody;
worker_processes  1;

#error_log  logs/error.log;
#error_log  logs/error.log  notice;
#error_log  logs/error.log  info;

#pid        logs/nginx.pid;
```

- **`user`**：指定运行 Nginx 服务的用户和用户组，默认为 `nobody`，在生产环境建议设置为一个非特权用户，提升安全性。
- **`worker_processes`**：指定工作进程数，通常设为 CPU 核心数以充分利用多核资源。
- **`error_log`**：定义错误日志的位置及日志级别（如 `notice`、`info`、`debug`），便于排查问题。
- **`pid`**：记录 Nginx 主进程的 PID 文件路径，用于管理进程。

---

### 6.2 `events` 块

```
events {
    worker_connections  1024;
}
```

- **`worker_connections`**：每个工作进程可以处理的最大连接数。理论最大连接数 = `worker_processes` × `worker_connections`，实际情况还受限于系统配置。

---

### 6.3 `http` 块

`http` 块是 Nginx 配置的核心部分，包含了与 HTTP 服务相关的配置和服务器定义。  
 通用配置

```
http {
    include       mime.types;
    default_type  application/octet-stream;

    #log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
    #                  '$status $body_bytes_sent "$http_referer" '
    #                  '"$http_user_agent" "$http_x_forwarded_for"';

    #access_log  logs/access.log  main;

    sendfile        on;
    #tcp_nopush     on;

    #keepalive_timeout  0;
    keepalive_timeout  65;

    #gzip  on;
}
```

- **`include mime.types`**：引入 MIME 类型配置文件，用于识别资源类型（如 HTML、CSS、JS）。
- **`default_type`**：未指定 MIME 类型时的默认值，设置为 `application/octet-stream` 表示二进制流。
- **`log_format` 和 `access_log`**：定义访问日志格式和存储路径，日志分析是运维的重要工具。
- **`sendfile`**：启用高效文件传输模式，用于优化静态文件的传输性能。
- **`keepalive_timeout`**：设置 HTTP 长连接的超时时间，默认为 65 秒。
- **`gzip`**：启用响应内容的压缩，提升带宽利用率。

---

### 6.4 `server` 块

`server` 块定义了虚拟主机的配置，是实际提供服务的单元。

```
server {
    listen       80;
    server_name  localhost;

    location / {
        root   html;
        index  index.html index.htm;
    }

    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   html;
    }
}
```

- **`listen`**：指定监听的端口号，此处设置为 `80`（HTTP 默认端口）。
- **`server_name`**：定义主机名或域名，此处为 `localhost`，可用于区分不同的虚拟主机。
- **`location`**：配置路径处理规则：
  - `/` 表示根路径，所有请求将映射到 `html` 目录下。
  - **`root`**：定义资源的根目录。
  - **`index`**：定义默认的首页文件名。
- **`error_page`**：自定义错误页面，例如 500 系列错误会跳转到 `/50x.html`。

---

### 6.5 特殊用途配置示例

1. **反向代理**

   ```
   location /api/ {
       proxy_pass http://127.0.0.1:8080;
   }
   ```

   将 `/api/` 请求转发到后端服务 `http://127.0.0.1:8080`。
2. **禁止访问某类文件**

   ```
   location ~ /\.ht {
       deny all;
   }
   ```

   禁止访问以 `.ht` 开头的文件。

### 6.6 其他服务器配置（HTTPS 支持）

```
server {
    listen       443 ssl;
    server_name  localhost;

    ssl_certificate      cert.pem;
    ssl_certificate_key  cert.key;

    ssl_session_cache    shared:SSL:1m;
    ssl_session_timeout  5m;

    ssl_ciphers  HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers  on;

    location / {
        root   html;
        index  index.html index.htm;
    }
}
```

- **`listen 443 ssl`**：启用 HTTPS 服务。
- **`ssl_certificate` 和 `ssl_certificate_key`**：指定 SSL 证书和私钥文件路径。
- **`ssl_session_cache` 和 `ssl_session_timeout`**：配置 SSL 会话缓存和超时。
- **`ssl_ciphers`**：指定加密算法，保障数据传输安全。
